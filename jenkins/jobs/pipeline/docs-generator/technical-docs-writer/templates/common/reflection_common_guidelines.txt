# 技術ドキュメント作成の共通ガイドライン

## セクション間の責任分離
- 各セクションは独自の専門領域に集中し、内容の重複を避ける
- 変更履歴情報は「変更履歴」セクションのみで管理

## セクション固有の責任
- 概要: システム全体の目的と主要機能の簡潔な説明
- ディレクトリ構造: フォルダ構成とファイル説明のみ
- システムアーキテクチャ図: コンポーネント関係図と設計原則のみ
- データフロー図: データの流れの視覚化と説明のみ
- 用語集: 専門用語の定義のみ
- 変更履歴: すべての変更記録

## 技術文書作成の原則
- 客観的かつ正確な記述
- ユーザー視点での説明
- 情報の整理と構造化
- 一貫性のある表現
- 視覚的な明瞭さ
- 現在形での説明

## マークダウン記法のガイドライン
- 見出しは階層に応じて適切なレベル(#, ##, ###)を使用
- コードブロックは```で適切に囲む
- テーブルは列の意味が明確になるよう設計
- 箇条書きは関連項目のグループ化に使用
- 強調は**太字**や*イタリック*を適切に使用
