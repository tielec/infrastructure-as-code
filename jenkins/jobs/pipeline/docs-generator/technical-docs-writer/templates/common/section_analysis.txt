## 利用可能なセクション
以下のセクションタイプから、PRの内容に関連するものを特定してください：
{section_types}

## 分析の観点

1. **変更内容の種類**
   - 機能の追加/変更/削除
   - アーキテクチャや構造の変更
   - 設定やデータフローの変更
   - ファイルやディレクトリ構造の変更
   - プロセスやワークフローの変更

2. **キーワードの出現**
   - セクションに特有のキーワードの有無
   - 技術専門用語の文脈
   - コンポーネント名や機能名への言及

3. **コードやダイアグラムの存在**
   - コードサンプルの種類と内容
   - mermaidなどのダイアグラム
   - ディレクトリ構造やファイル一覧

## 回答フォーマット

以下のJSONフォーマットで関連するセクションを返してください：

```json
{
  "related_sections": ["セクション1", "セクション2", ...],
  "rationale": "これらのセクションが関連する理由の簡潔な説明"
}
```

注意: related_sectionsは利用可能なセクションから選択し、完全に一致する文字列を使用してください。
