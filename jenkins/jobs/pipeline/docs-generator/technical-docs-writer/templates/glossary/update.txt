## 重要: PRから抽出された技術用語のみを使用

用語集の更新には、PRコメントや既存の用語集に実際に言及されている技術用語のみを含めてください。このプロンプトで参照されている例や説明用の用語は含めないでください。

## 更新の観点

1. **新しい用語の追加**
   - PRで言及された新しい一般的な技術用語や概念のみ
   - PRコメントに言及されている標準的な技術スタックに関連する新しい用語のみ

2. **既存の用語の更新**
   - 定義の明確化や精緻化
   - 技術的に正確な情報への更新
   - 追加情報や補足説明の提供

3. **不適切な用語の削除**
   - ファイル名やスクリプト名の削除
   - ジョブ名やタスク名の削除
   - プロジェクト固有の用語やカスタム命名の削除

4. **「プロジェクト固有の専門用語」カテゴリの除去**
   - このカテゴリが存在する場合は完全に削除
   - このカテゴリ内の用語は、一般的な技術用語であれば適切なカテゴリに移動
   - プロジェクト固有の用語は削除

### 用語として含めるべきもの・含めるべきでないもの

#### 含めるべきもの
- PRコメントで言及されている技術的な概念や用語
- PRコメントで言及されている一般的なサービスやツールの名称
- PRコメントで言及されている業界標準のアーキテクチャパターンや設計概念
- PRコメントで言及されている広く使用されている技術標準や規格

#### 含めるべきでないもの
- ファイル名やスクリプト名
- 具体的なファイルパス
- 一般的なコマンド
- ジョブ名やタスク名
- プロジェクト固有の命名や略語
- コミットID、PRナンバーなどの一時的な識別子
- プロジェクト固有の用語・概念
- このプロンプト内の例示や説明用の用語

### 「プロジェクト固有の専門用語」カテゴリの取り扱い

既存の用語集に「プロジェクト固有の専門用語」カテゴリが含まれている場合は、以下のように処理してください：

1. このカテゴリ全体を削除する
2. このカテゴリ内の用語のうち：
   - 一般的な技術用語に該当するものは、適切な技術カテゴリに移動
   - ファイル名、スクリプト名、ジョブ名、プロジェクト固有の用語は削除

### 重要: テンプレートの例示は含めない

このプロンプトのいかなる部分も実際の出力に含めないでください。あくまでPRコメントと既存の用語集から抽出された実際の用語のみを使用してください。このプロンプトに記載されているカテゴリ名や用語の例は、形式の参考としてのみ使用し、実際の出力には含めないでください。

### カテゴリ構造の改善ガイドライン

カテゴリ構造の改善は以下の指針で行ってください：

1. **技術/サービス別の具体的カテゴリを使用**
   - PRコメントとプロジェクトの実際の技術スタックに基づく明確な技術領域ごとに分類
   - 実際のプロジェクトの技術に基づく具体的な技術名をカテゴリ名として使用

2. **関連する用語を同じカテゴリに集約**
   - 関連する技術用語は同じカテゴリにまとめる
   - 必要に応じて新しいカテゴリを作成し、関連する用語を移動

3. **バランスのとれたカテゴリ構造の維持**
   - 各カテゴリに少なくとも2つ以上の用語を含める
   - 1つの用語のためだけにカテゴリを作成しない
   - 用語が多すぎるカテゴリは、より細分化することを検討

## フォーマットのガイドライン

既存の形式を基本的に維持してください：

```markdown
## 用語集

### [PRから抽出されたカテゴリ名]
- **[PRから抽出された用語1]**: [その用語の説明]
- **[PRから抽出された用語2 (正式名称)]**: [その用語の説明]

### [PRから抽出された別のカテゴリ名]
- **[PRから抽出された用語3]**: [その用語の説明]
- **[PRから抽出された用語4]**: [その用語の説明]
```

## 用語定義のガイドライン

各用語の定義は以下の特性を備えるようにしてください：

- **簡潔さ**: 1-2文で概念を説明
- **明確さ**: 技術的に正確で具体的
- **一般性**: 業界で広く使用される一般的な定義
- **一貫性**: 同様の用語に対して同様のスタイルで説明
- **完全性**: 略語の場合は正式名称を含める

## 出力の要件

- 更新された用語集セクション全体をMarkdownで出力
- 見出しは「# 用語集」で始める
- カテゴリごとにサブ見出し（###）を使用
- カテゴリ内では用語をアルファベット順に並べる
- 既存の形式（箇条書きリスト）を維持
- 各用語は太字で強調
- 略語は正式名称を括弧内に記載

更新の際は、プロジェクト固有の専門用語カテゴリを完全に削除し、一般的な技術用語のみを含めるように修正してください。ファイル名、スクリプト名、ジョブ名は用語として含めないでください。

このプロンプト自体は参考用であり、ここに含まれる説明や例示は最終出力には含めないでください。PRコメントと既存の用語集から実際に必要な用語のみを含めてください。
