Private Keyを生成してダウンロード\n   182→4. 秘密鍵をPKCS#8形式に変換してSSMに手動登録:\n   183→\n   184→```bash\n   185→# PKCS#8形式に変換（Jenkinsで必要）\n   186→# GitHubからダウンロードした鍵はPKCS#1形式（BEGIN RSA PRIVATE KEY）\n   187→# JenkinsにはPKCS#8形式（BEGIN PRIVATE KEY）が必要\n   188→openssl pkcs8 -topk8 -inform PEM -outform PEM -nocrypt \\\n   189→  -in github-app-key.pem \\\n   190→  -out github-app-key-pkcs8.pem\n   191→\n   192→# SSMパラメータストアに登録\n   193→aws ssm put-parameter \\\n   194→  --name "/bootstrap/github/app-private-key" \\\n   195→  --value file://github-app-key-pkcs8.pem \\\n   196→  --type SecureString \\\n   197→  --overwrite \\\n   198→  --region ap-northeast-1\n   199→```\n   200→\n   201→**SSMパラメータ**:\n   202→- `/bootstrap/github/app-id` - App ID\n   203→- `/bootstrap/github/app-private-key` - 秘密鍵（要手動登録）\n   204→- `/bootstrap/github/app-owner` - 組織名（オプション）\n   205→\n   206→##### 4. Pulumiパスフレーズの設定（手順9で実行）\n   207→\n   208→**初回実行時**:\n   209→- 自動生成または手動入力を選択\n   210→- SSMパラメータストアに暗号化保存\n   211→\n   212→**重要な注意事項**:\n   213→- **一度設定したパスフレーズは変更不可**（既存スタックへのアクセスが失われる）\n   214→- バックアップを強く推奨\n   215→\n   216→**バックアップ方法**:\n   217→```bash\n   218→# パスフレーズを取得して安全な場所に保存\n   219→aws ssm get-parameter \\\n   220→  --name "/bootstrap/pulumi/config-passphrase" \\\n   221→  --with-decryption \\\n   222→  --query \'Parameter.Value\' \\\n   223→  --output text\n   224→```\n   225→\n   226→**SSMパラメータ**:\n   227→- `/bootstrap/pulumi/config-passphrase` - パスフレーズ（SecureString）\n   228→\n   229→##### 設定値の永続性\n   230→\n   231→すべての設定はSSMパラメータストアに保存されるため：\n   232→- EC2インスタンスを再作成しても設定が保持される\n   233→- 2回目以降の実行では自動的に復元される\n   234→- 手動作業が必要なのは初回のみ（GitHub公開鍵登録、GitHub App秘密鍵登録）\n   235→\n   236→### 4. Pulumiバックエンドの設定\n   237→\n   238→本プロジェクトはデフォルトでS3バックエンドを使用してPulumiの状態を管理します。\n   239→\n   240→#### S3バックエンドの設定（推奨）\n   241→\n   242→S3バックエンドはCloudFormationブートストラップで作成されたS3バケットを使用します。パスフレーズは`setup-bootstrap.sh`実行時に対話形式で設定され、SSM Parameter Storeに安全に保存されます。\n   243→\n   244→##### 初回セットアップ\n   245→\n   246→`setup-bootstrap.sh`を実行すると、以下の処理が自動的に行われます：\n   247→\n   248→1. **S3バケットの確認**: CloudFormationで作成されたバケットを自動検出\n   249→2. **パスフレーズの設定**: 対話形式で設定（自動生成または手動入力を選択可能）\n   250→3. **SSM Parameter Storeへの保存**: SecureStringタイプで暗号化して保存\n   251→\n   252→##### Ansible実行時の自動設定\n   253→\n   254→**重要**: Ansibleは自動的にSSM Parameter Storeからパスフレーズを取得するため、通常は環境変数の設定は不要です。\n   255→\n   256→```bash\n   257→# Ansibleを実行（パスフレーズは自動的にSSMから取得）\n   258→cd ansible\n   259→ansible-playbook playbooks/jenkins/jenkins_setup_pipeline.yml -e "env=dev"\n   260→```\n   261→\n   262→##### 手動での環境変数設定（オプション）\n   263→\n   264→環境変数を優先したい場合や、SSMへのアクセスを避けたい場合：\n   265→\n   266→```bash\n   267→# SSMからパスフレーズを取得して環境変数に設定\n   268→export PULUMI_CONFIG_PASSPHRASE=$(aws ssm get-parameter \\\n   269→  --name "/bootstrap/pulumi/config-passphrase" \\\n   270→  --with-decryption \\\n   271→  --query \'Parameter.Value\' \\\n   272→  --output text)\n   273→\n   274→# S3バケット名はSSMパラメータストアから自動取得されるため、手動設定は不要\n   275→```\n   276→\n   277→**パスフレーズの優先順位**:\n   278→1. 環境変数 `PULUMI_CONFIG_PASSPHRASE`（設定されている場合）\n   279→2. SSM Parameter Store `/bootstrap/pulumi/config-passphrase`（自動取得）\n   280→3. エラー（どちらも利用できない場合）\n   281→\n   282→##### パスフレーズの管理\n   283→\n   284→- **確認**: `aws ssm get-parameter --name "/bootstrap/pulumi/config-passphrase" --with-decryption --query \'Parameter.Value\' --output text`\n   285→- **変更**: SSMコンソールまたはCLIで直接更新（既存のPulumiスタックがある場合は注意）\n   286→- **セキュリティ**: SSM Parameter StoreでKMS暗号化されているため安全\n   287→\n   288→**重要**: パスフレーズは一度設定したら変更しないでください。変更すると既存のPulumiスタックにアクセスできなくなります。\n   289→\n   290→### 5.
