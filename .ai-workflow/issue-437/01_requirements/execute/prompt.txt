### 本文

## 概要
Jenkins Agentのメモリ使用状況をCloudWatchで監視できるようにする。
アラート設定は不要で、メトリクスの収集と可視化のみを実装する。

## 背景
Jenkins Agentの安定稼働のため、メモリ使用状況を監視する必要がある。
CloudWatch Agentを使用してメモリメトリクスを収集し、CloudWatchコンソールで確認できるようにする。

## 実装内容

### 1. Pulumi: IAM権限の追加
- `pulumi/jenkins-agent-autoscaling/` のIAMロールにCloudWatch Agent用の権限を追加
  - `CloudWatchAgentServerPolicy` マネージドポリシーをアタッチ

### 2. Ansible: CloudWatch Agentのセットアップ
- `ansible/roles/jenkins_agent/` にCloudWatch Agent設定を追加
  - CloudWatch Agentのインストール
  - メモリメトリクス収集設定ファイルの配置
  - CloudWatch Agentサービスの起動・有効化

### 3. 収集するメトリクス
- メモリ使用率（パーセント）
- メモリ使用量（MB）
- メモリ空き容量（MB）

### 4. ⚠️ コスト最適化設定（重要）
スポットインスタンスで頻繁にインスタンスが入れ替わる環境のため、以下の設定で実装する：

**Dimensionは `AutoScalingGroupName` のみを使用**
- インスタンスIDを含めない（デフォルトは含まれる）
- インスタンスが何回入れ替わってもメトリクス数が増えない
- ASG全体の平均値として監視

**コスト比較:**
```
❌ インスタンスIDを含める場合（デフォルト）:
  - 10回入れ替わり → 20メトリクス → 約$6/月

✅ AutoScalingGroupNameのみの場合（推奨）:
  - 何回入れ替わっても → 2-3メトリクス → 約$0.60-1.0/月
```

**CloudWatch Agent設定例:**
```json
{
  "metrics": {
    "namespace": "CWAgent",
    "metrics_collected": {
      "mem": {
        "measurement": [
          {"name": "mem_used_percent"},
          {"name": "mem_used"},
          {"name": "mem_available"}
        ]
      }
    },
    "append_dimensions": {
      "AutoScalingGroupName": "${aws:AutoScalingGroupName}"
    }
  }
}
```

## 品質ゲート（Phase 1）

作成する要件定義書は、以下の品質ゲートを満たす必要があります：

- [ ] **機能要件が明確に記載されている**
- [ ] **受け入れ基準が定義されている**
- [ ] **スコープが明確である**
- [ ] **論理的な矛盾がない**

これらの品質ゲートは**必須要件**です。作成後、クリティカルシンキングレビューが実施され、品質ゲートを満たさない場合は修正が必要になります。

## 注意事項

1. **具体性**: 曖昧な表現を避け、具体的かつ測定可能な要件を記述
   - NG例: "適切に処理する"、"必要に応じて対応"
   - OK例: "3秒以内にレスポンスを返す"、"エラー発生時は管理者に通知"

2. **整合性**: 各セクション間で矛盾がないか確認
   - 機能要件と受け入れ基準が対応しているか
   - 非機能要件と制約事項が矛盾していないか

3. **完全性**: Issue本文の情報を漏れなく反映
   - TODOセクションの項目をすべて機能要件に反映
   - 概要セクションの背景・目的を明確に記述

4. **検証可能性**: 各要件が検証可能（テスト可能）な形で記述
   - 受け入れ基準は Given-When-Then 形式で記述
   - 成功条件・失敗条件が明確に識別可能

5. **レビューされることを意識**: 作成した要件定義書はクリティカルシンキングレビューが実施されます
   - 品質ゲート（4つの必須要件）を最優先で満たすこと
   - ブロッカー（次フェーズに進めない問題）がないように注意
   - 改善提案は歓迎されるが、ブロッカーは修正が必須
