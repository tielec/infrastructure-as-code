## 更新詳細

### 1. jenkins/README.md

**更新箇所**: 行100-103

**更新理由**:
初期セットアップ手順において、シードジョブの実行方法が実装変更により変わったため、ユーザー向けドキュメントを更新しました。

**変更内容**:
- **変更前**: 「`job-creator` シードジョブを実行すると、AI Workflowジョブは自動的に作成されます」と記載
- **変更後**: 「2つのシードジョブを実行：`job-creator`（一般ジョブを生成）と`ai-workflow-job-creator`（AI Workflowジョブを生成）」に更新

**影響範囲**:
- Jenkins初期セットアップを行うユーザー全員
- 新規環境構築時の手順

**具体的な変更**:
```markdown
# 変更前
# 3. シードジョブの実行
# Jenkins UIから Admin_Jobs > job-creator を実行
# AI Workflowジョブは自動的に作成されます

# 変更後
# 3. シードジョブの実行
# Jenkins UIから以下の2つのシードジョブを実行：
# - Admin_Jobs > job-creator（一般ジョブを生成）
# - Admin_Jobs > ai-workflow-job-creator（AI Workflowジョブを生成）
```

### 2. jenkins/CONTRIBUTION.md

**更新箇所**: 行197-204

**更新理由**:
実装により設定ファイルが完全に分離されたため、開発者向けドキュメントにこの重要なアーキテクチャ変更を反映しました。

**変更内容**:

**影響範囲**:
- Jenkins Job DSLを編集する開発者
- シードジョブのアーキテクチャを理解する必要がある開発者
- 新しいシードジョブを追加する開発者

**具体的な変更**:
```markdown
# 追加した内容
- **設定ファイルの完全分離**: 専用の`job-config.yaml`と`folder-config.yaml`により、AI Workflowと一般ジョブが完全に独立
```
